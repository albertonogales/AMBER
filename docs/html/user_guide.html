Training a SOM

Basic usage

import numpy as np
import AMBER

data = np.random.rand(300, 10)   # 300 samples, 10 features

som = AMBER.Map(data=data, period=200)
# Map size is chosen automatically by the Vesanto & Alhoniemi heuristic:
#   size = max(2, round(sqrt(5 * sqrt(N))))
print(f"Map size: {som.map_size}×{som.map_size}")
What does period mean? period is the total number of individual pattern presentations, not the number of full passes over the dataset. With period=200 and N=300 samples, only ~67% of the data is presented on average (in random mode). To approximate K full epochs, set period = K × N. For example, 10 epochs over 300 samples → period=3000.

Choosing the map size manually

som = AMBER.Map(data=data, size=10, period=200)

# The Vesanto helper is also available directly:
print(AMBER.vesanto_size(300))   # recommended side length for 300 samples

All training options

som = AMBER.Map(
    data=data,
    period=500,
    size=0,               # 0 = auto (Vesanto heuristic)
    initial_lr=0.5,       # starting learning rate
    initial_neighbourhood=0,  # 0 = auto-set to map_size
    use_decay=True,       # cosine decay of lr and neighbourhood
    distance='euclidean', # BMU search distance
    normalization='zscore',
    weights='PCA',        # weight initialisation
)

Reproducibility

By default, AMBER initialises weights and selects training samples using the global NumPy RNG, so results vary between runs. Pass random_seed to get fully reproducible training:

som = AMBER.Map(data=data, size=10, period=200, random_seed=42)

# Two maps with the same seed produce identical weights:
som1 = AMBER.Map(data=data, size=10, period=200, random_seed=42)
som2 = AMBER.Map(data=data, size=10, period=200, random_seed=42)
import numpy as np
np.testing.assert_array_equal(som1.weights, som2.weights)  # passes

# The seed is persisted in save/load:
som1.save_classifier('my_model')
loaded = AMBER.Map.load_classifier('my_model')
print(loaded.random_seed)  # 42

TemporalMap accepts random_seed in the same way and passes it through to the parent Map.

Distance Metrics

NameWhen to use
euclidean (default)General-purpose; fast; works well on normalised data
manhattanRobust to outliers in individual dimensions
chebyshevSensitive to the single largest dimension difference
cosinePattern shape matters more than magnitude (text, gene expression)
correlationAmplitude-invariant; standard in climatology and EEG
dtwTime series with phase/speed variation (speech, vibration, ECG)
cross_correlationShift-invariant similarity between periodic signals
 
som = AMBER.Map(data=data, period=200, distance='dtw')
print(AMBER.AVAILABLE_DISTANCES)   # full list

Classification

cls = AMBER.Classification(som, data)

# Quality metrics
print(cls.quantization_error)           # primary: uses the map's configured distance
print(cls.quantization_error_euclidean) # secondary: always Euclidean (cross-library comparison)
print(cls.distortion)                   # Graepel–Heskes distortion (vectorised)
print(cls.topological_error)            # fraction with non-adjacent 2nd BMU

# Per-sample BMU positions — DataFrame with columns x, y, bmu_distance
print(cls.classification_map.head())

# Activation map: (map_size, map_size) count of BMU hits
print(cls.activations_map)

# U-matrix: (2k-1, 2k-1) inter-neuron distance grid
print(cls.umatriz.shape)
Metrics note: quantization_error uses the same distance metric configured for BMU search (e.g. 'manhattan', 'cosine', etc.) — this is the primary, semantically correct metric. quantization_error_euclidean always uses L2 regardless of the training distance, which makes it comparable with results from other SOM libraries. distortion (Graepel & Heskes 1997) sums Gaussian-weighted squared distances from each sample to all neurons — a global measure of fit.

Labelled classification

labels = np.array(['A', 'B', 'A', ...])   # one label per sample
cls    = AMBER.Classification(som, data, labels)

Temporal / Recurrent SOM

TemporalMap extends Map with a context vector (Voegtlin 2002 RSOM):

  • Context update: ct = α · ct−1 + (1−α) · wBMU
  • Effective distance: deff = (1−β) · dsignal + β · dcontext

where α is context_weight (memory decay) and β is context_influence.

tsom = AMBER.TemporalMap(
    data=data,
    period=200,
    context_weight=0.6,     # α — how much history is retained
    context_influence=0.3,  # β — how much context affects BMU selection
    distance='euclidean',
    normalization='zscore',
)

tcls = AMBER.Classification(tsom, data)
ta   = AMBER.TemporalAnalysis(tcls)

print(ta.summary())
# stability        — fraction of consecutive identical BMUs
# mean_path_length — mean Euclidean grid distance per step

# Most frequent transitions
for (src, dst), count in ta.most_frequent_transitions(top_k=5):
    print(f"neuron {src} → {dst}  ({count} times)")

# Reset context between independent sequences
tsom.reset_context()

Feature Extraction

from AMBER import FeatureExtractor

fe = FeatureExtractor(fs=256)   # 256 Hz sampling rate

# Single window → 1-D feature vector
x = fe.extract(signal_window, features=[
    'rms', 'zero_crossing_rate',
    'alpha_power', 'beta_power', 'spectral_entropy',
    'hjorth_mobility', 'hjorth_complexity',
])

# Batch → (n_windows, n_features) array ready for Map
X = fe.extract_batch(windows, features=['rms', 'kurtosis', 'spectral_entropy'])

Iterative Map-Size Selection

IterativeSOM trains maps across a range of sizes and returns the one with the lowest topological error:

isom = AMBER.IterativeSOM(data=data, period=100)
best = isom.best_map()
print(f"Best size: {best.map_size}")

# Inspect all sizes tested
print(isom.calculate_range())

Save and Load

# Save to JSON (no pickle — human-readable)
som.save_classifier('my_model')          # writes my_model.json

# Load back
loaded = AMBER.Map.load_classifier('my_model')
_, bmu_pos, _, _ = loaded.calculate_bmu(data[0])

# TemporalMap save/load works the same way
tsom.save_classifier('temporal_model')
loaded_t = AMBER.TemporalMap.load_classifier('temporal_model')
