Installation

AMBER requires Python 3.9 or later. It is tested on Linux, macOS, and Windows.

The package is distributed via PyPI using the modern pyproject.toml (PEP 517/518) build system — no legacy setup.py required.

Requirements

The core library depends only on well-established scientific Python packages:

PackageMinimum versionPurpose
numpy≥ 1.21Array operations and linear algebra throughout
pandas≥ 1.3Classification map and temporal analysis DataFrames
matplotlib≥ 3.5All static visualisations
plotly≥ 5.0Interactive heat maps and U-matrix
scipy≥ 1.7PCA weight initialisation, spectral features
scikit-learn≥ 1.0Normalisation helpers

Core Install

pip install amber-som

Optional Extras

Spectral features

Required only when using FeatureExtractor with frequency-domain features (spectral_power, dominant_frequency, EEG band powers, etc.):

pip install "amber-som[spectral]"

MFCC features

Required only when using FeatureExtractor with mfcc:

pip install "amber-som[mfcc]"
# installs librosa and soundfile

All extras at once

pip install "amber-som[spectral,mfcc]"

Install from Source

git clone https://github.com/<your-org>/amber-som.git
cd amber-som
pip install -e .
Notebooks: each example notebook contains a path-fix cell at the top so it works even without pip install — just open it from the examples/ folder.

Verifying Your Install

import AMBER
print(AMBER.__version__)          # 2.1.0
print(AMBER.AVAILABLE_DISTANCES)  # ['euclidean', 'manhattan', ...]

Developer Install

# 1. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# 2. Install with dev dependencies
pip install -e ".[dev]"
# — or —
pip install -r requirements-dev.txt && pip install -e .

# 3. Run the test suite
pytest                                        # 315 tests
pytest --cov=AMBER --cov-report=term-missing  # with coverage report
Tip: All 315 tests use synthetic data — no real datasets or internet access required.
