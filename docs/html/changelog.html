Changelog

All notable changes to AMBER are documented here.

v2.1.0 — 2026

Added

  • random_seed parameter on Map and TemporalMap — pass an integer for fully reproducible training; persisted in JSON save/load
  • Type annotations on all public methods in map.py, temporal_map.py, and classification.py
  • ruff and mypy added to CI and dev dependencies — linting and type-checking enforced on every push
  • 12 new tests: TestReproducibility, TestReinforce, TestSaveLoadExtended, TestTemporalMapReproducibility (315 tests total, 96% coverage)

Changed

  • All print() calls replaced with logging (logger.info/logger.debug) — libraries no longer pollute stdout
  • Packaging consolidated to a single pyproject.toml (PEP 517/518); setup.py removed; requires-python = ">=3.9"
  • Classification.__init__ verbose parameter changed from int to bool (verbose: bool = False)
  • Vectorised distortion and Euclidean QE computation in classification.py — significant speedup for large datasets
  • All RNG calls in map.py use instance-level np.random.default_rng(random_seed) — no global state mutation
  • CI matrix updated to Python 3.9–3.12 (dropped EOL 3.8; added 3.12)
  • Input validation changed from assert to raise ValueError — guards are not disabled by python -O
  • period docstring clarified: it counts total pattern presentations, not full epochs (period = K × N ≈ K epochs)
  • reinforce() neighbourhood now decays correctly using variation_neighbourhood rather than being hardcoded to 1
  • Removed duplicate self.presentation assignment in Map.__init__
  • cross_correlation_distance docstring updated to explain why the [0,1] bound holds (Cauchy-Schwarz on unit-norm sub-vectors)

Fixed

  • reinforce() now correctly passes mode=self.lr_decay to variation_learning_rate
v2.1.1 — 2026

Added

  • TemporalAnalysis: two new scalar attributes — temporal_coherence (fraction of consecutive BMU steps with Chebyshev distance ≤ 1; range [0, 1]) and mean_chebyshev_jump (mean L∞ grid distance per step). Both are computed in __init__ and reported by summary(). transition_matrix_norm (row-normalised transition probabilities) is also now exposed as a public attribute.
  • Visualization: three new static methods —
    • umatrix_labeled(cls, labels, …) — U-matrix background with coloured circle markers showing the majority class and sample count per neuron; legend placed between title and axes
    • hit_map(cls, labels, …) — hit map where cell size scales with √(n/n_max) and background tint encodes the majority class; legend placed above axes with up to 5 columns
    • weight_map_grid(som, cls, labels, …) — k×k subplot grid, one line-plot per neuron's weight vector, background tinted by majority class, dead neurons shown in grey with address annotation
v2.0.0 — 2024

Added

  • Renamed library from GEMA/AMBAR to AMBER (Autoassociative Map Builder for tEmporal Representations)
  • PyPI package name amber-som
  • TemporalMap — Recurrent SOM with context vector (Voegtlin 2002)
  • TemporalAnalysis — transition matrix, dwell times, trajectory, stability metrics
  • FeatureExtractor — 20+ features for biosignals and audio (band powers, Hjorth, MFCC, entropy)
  • IterativeSOM — automatic map-size selection by topological error
  • Seven signal distance metrics: euclidean, manhattan, chebyshev, cosine, correlation, DTW, cross-correlation
  • Eight normalisation strategies including per-sample variants
  • Four weight-initialisation strategies: random, random-negative, sample, PCA
  • Cosine decay of learning rate and neighbourhood radius
  • JSON save/load for Map and TemporalMap (no pickle)
  • Visualisation: heat map, U-matrix, elevation map, codebook vectors, trajectory, transition matrix, dwell-time map
  • 265 pytest tests, 95 % coverage
  • Sphinx HTML documentation
  • CI via GitHub Actions (Python 3.8 – 3.11)
  • Example notebooks: EEG, audio, climate, predictive maintenance, ablation study, library comparison

Changed

  • All public classes re-documented with full NumPy-style docstrings
  • Package structure aligned with PEP 517/518 (pyproject.toml)

Fixed

  • Colourmap resampling call compatible with Matplotlib ≥ 3.7
  • RST pipe-character conflict in distances.py docstring
v1.0.0 — GEMA

Original internal library (GEMA / AMBAR). Basic SOM training and classification, Euclidean distance only.
