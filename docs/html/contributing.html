Reporting Bugs

Open an issue on GitHub and include:

  • AMBER version (AMBER.__version__)
  • Python version and OS
  • Minimal reproducible example
  • Full traceback

Requesting Features

Open a GitHub issue with the label enhancement. Describe the use case and, if possible, the expected API.

Development Setup

git clone https://github.com/<your-org>/amber-som.git
cd amber-som
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
pytest   # all 265 tests should pass

Submitting a Pull Request

  1. Fork the repository and create a feature branch: git checkout -b feature/my-feature
  2. Write tests for your changes — coverage should not decrease
  3. Run pytest and confirm all tests pass
  4. Add a changelog entry in CHANGELOG.md
  5. Open a pull request against main

Adding a Distance Metric

All distances live in AMBER/distances.py. To add my_distance:

  1. Add a scalar function my_distance(a, b) → float
  2. Add a matrix function my_distance_matrix(weights, pattern) → ndarray (rows, cols)
  3. Register both in SIGNAL_DISTANCE_MATRIX dict
  4. Add tests in tests/test_distances.py
# AMBER/distances.py

def my_distance(a, b):
    ...

    ...

SIGNAL_DISTANCE_MATRIX['my_distance'] = my_distance_matrix

Adding a Normalisation Strategy

Normalisation is handled inside Map.__normalize(data, method) in AMBER/map.py. Add a new elif method == 'my_norm' branch and update the error message listing available strategies.
